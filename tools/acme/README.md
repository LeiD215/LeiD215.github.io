# 📜 基于 acme.sh 的全自动 SSL 证书申请与多域名管理一键工具说明

本工具是基于 `acme.sh` 内核进行生产级韧性硬化重构的一键部署脚本。工具默认在 `set -uo pipefail` 严格模式下运行，全量显式宣告并初始化了所有交互变量，从底层根除了由于环境波动、未定义变量、网络断裂、或人工中断（`Ctrl + C`）引发的进程死锁与闪退隐患。

---

## 🛠️ 1. 核心功能模块解析

* **多域名独立建档与分流保存**
* **业务逻辑**：打破了原脚本只能管理单个域名的限制。每申请一个新域名，脚本都会自动在 `/root/cert/各自的域名/` 下创建一个专属文件夹，并将证书文件独立规范化导出为 `fullchain.pem` 和 `privkey.pem`。多个域名同时并存，互不干扰、永不覆盖。
* **高可用硬化**：重构了证书配对验证机制。废除了传统易在现代现代 ECC 证书上引发逻辑死锁的 `openssl -modulus`（模数）校验，全面升级为提取公私钥 **DER 二进制摘要指纹 (SHA-256)** 的双向暗号验证。不论是 RSA 还是 ECC 算法，只有两者的摘要完全配对，才会放行独立建档与分流导出，严防损坏的证书混入文件系统。

* **两种证书申请模式（自动处理双栈网络）**
* **独立 80 端口模式 (Standalone)**：适合纯净系统。引入了**端口释放状态机**，检测到 80 端口被占用时平滑停止 `Nginx/Apache2/Caddy` 并记录其原有运行状态。自动识别纯 IPv6、纯 IPv4 或双栈网络执行条件引导。证书申请结束后，会自动还原拉起原有的 Web 服务，确保业务零中断。
* **DNS API 模式**：适合生产环境，支持 Cloudflare、腾讯云 DNSPod、阿里云 Aliyun。此模式下完美支持单域名与泛域名（如 `*.domain.com`）的申请。由于不需要占用 80 端口，申请和续期时完全不影响服务器上现有网站的正常运行。
* **前端交互硬化**：彻底修复了一级菜单（模式选择）和二级菜单（解析商选择）中，因旧版 `read -p` 不支持 `\n` 转义导致选项挤在一行的排版缺陷。全面改用标准 `printf` 换行渲染，让 `请选择：` 提示符独立留在最后一行最左侧顶头，交互井然有序。局部业务函数内注入了具名异常捕捉器 (`trap ... INT TERM ERR`)，杜绝了用户在输入 API 密钥阶段因 `Ctrl + C` 中断导致的菜单闪退，且退回菜单时 100% 擦除内存中残留的云商密钥，保障凭证零泄漏。

* **多证书智能全自动续期**
* **业务逻辑**：首次成功申请后，工具会向系统的 crontab 计划任务中写入一条每日凌晨 0 点执行的自动续期指令。由于重构版中把动态路径死死地注册进了 acme.sh 的底层配置中，所有托管的域名在快到期时，都会在后台自动排队续期并精准同步到各自的专属文件夹，实现真正的无人值守。
* **高可用硬化**：弃用了会导致 Crontab 定时任务文件在多域名高频重复调用下发生非幂等行堆积、增殖污染的模糊匹配，全面升级为基于**专属私有锚点 `#acme-yg-auto**` 的流式原子定向爆破与精准挂载。同时彻底清除了挂载任务中错误的 `root` 字段，严格对齐用户级 crontab 的标准时间格式，修复了自动续期因格式错误导致被 Linux cron 守护进程拒绝执行的底层缺陷。

* **智能防死锁（强制覆盖模式）**
* **业务逻辑**：自动检索 `acme.sh --list` 现存沙箱记录。如果检测到当前域名在系统内已有历史申请记录，会自动在底层追加 `--force` 参数，启动强制覆盖更新，方便随时更换解析商或重新生成密钥。
* **高可用硬化**：将危险的“全局字符串硬拼接”彻底重构为 Bash 标准的“动态参数数组隔离传参（`"${acme_args[@]}"`）”。配合最顶层对 `acme_args=()` 的高频定时主动清洗，彻底封堵了由于空格截断、变量二次展开（Word Splitting）导致的参数死锁与代码注入漏洞。

* **全局证书状态统计与查询**
* **主界面动态统计**：每次打开脚本，主界面都会调用 `wc -l` 动态计算并实时显示当前系统内一共成功托管了多少个有效证书。
* **一键列表查询**：提供专门的菜单选项，可以一键调用原厂 `acme.sh --list` 接口，完整罗列出所有已成功申请的域名、密钥长度、创建时间以及下一次自动续期的具体时间点。
* **高可用硬化**：主菜单完全收拢进安全的持久化 `while true` 循环中。解耦了 `acme` 函数与 `ACMEDNS` 函数中的 `choice` 局部同名作用域，断绝任何因信号异常中断返回持久循环时造成的变量污染，达成工业级生产环境高可靠闭环。

## 📋 2. 菜单选项功能及后台硬化逻辑一览

| 菜单编号 | 功能名称 | 对应后台底层硬化逻辑 |
| --- | --- | --- |
| **1** | **申请新证书** | 交互式输入域名。调用前置环境探测，引入 `icanhazip/ip.sb/ipify` **三重外网容灾集群轮询**秒级感知 VPS 真实 IP；解析 IP 采用 `dig` 校验，在无 `dig` 的极简容器环境下自动平滑**向后降级到 `nslookup` 匹配**；通过 `case` 精确流阻断空值与空值的意外对齐误判；网络高危安装管道包装在 `set +o pipefail` 的独立子 shell 中，严防网络超时造成的静默暴毙；签发成功后创建独立文件夹并使用 `printf` 规范化流式导出。 |
| **2** | **查看所有域名列表** | 统一了错误返回风格。剔除了原先会导致持久循环当场自杀的直接 `exit 1` 强杀指令，将功能包装为标准的控制流 `return 0`，无缝回传至主菜单循环，完整打印当前托管的所有证书信息明细。 |
| **3** | **手动强制一键续期** | 显式清洗动态传参参数，调用原厂 `acme.sh --cron -f` 后台核心续期引擎，忽略 90 天到期时限限制，立刻强行让系统内所有已注册域名在后台进行一次文件刷新与同步。 |
| **4** | **彻底清空并卸载** | 调用原厂 `--uninstall` 净化清洗引擎，撤销所有证书并强力清洗 `/root/cert` 目录及环境软连接；利用 `grep -v -F "#acme-yg-auto"` 干净抹除自建的计划任务并彻底御载 acme.sh 核心环境，最后安全退出系统。 |
| **0** | **退出** | 释放所有局部信号，干净退出当前 Shell 交互界面。 |
