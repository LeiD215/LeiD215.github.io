# 小白（就是我）自用的防火墙设置脚本

## Usage

First of all, download the main script:
```
{ curl -sSL https://github.com/LeiD215/LeiD215.github.io/raw/refs/heads/master/tools/Firewall/firewall.sh > firewall.sh || wget -qO firewall.sh https://github.com/LeiD215/LeiD215.github.io/raw/refs/heads/master/tools/Firewall/firewall.sh; } && chmod +x firewall.sh && sudo ./firewall.sh
```

### 1. 核心功能模块

* **多域名独立建档与分流保存**
打破了原脚本只能管理单个域名的限制。现在你每申请一个新域名，脚本都会自动在 `/root/cert/` 下创建一个以该域名命名的专属文件夹，并将证书文件重命名为 `域名.crt` 和 `域名.key` 存入其中。多个域名同时并存，互不干扰、永不覆盖。
* **两种证书申请模式（自动处理双栈网络）**
* **独立 80 端口模式（Standalone）**：适合纯净系统。脚本会自动检测 IPv4/IPv6 环境，若发现 80 端口被占用，会人性化地提示并强制释放端口，随后利用 Letsencrypt 签发 ECC 256 位高强度证书。
* **DNS API 模式**：适合生产环境。支持 **Cloudflare**、**腾讯云 DNSPod**、**阿里云 Aliyun**。此模式下完美支持单域名与泛域名（如 `*.domain.com`）的申请，且由于不需要占用 80 端口，申请和续期时完全不影响服务器上现有网站的正常运行。


* **多证书智能全自动续期**
脚本在首次成功申请证书后，会向系统的 `crontab` 计划任务中写入一条每日定时指令（凌晨 0 点执行）。因为我们在重构版中把动态路径死死地注册进了 `acme.sh` 的底层配置中，所以**所有托管的域名在快到期时，都会在后台自动排队续期，并各自精准刷新到对应的专属文件夹中**，实现真正的无人值守。
* **智能防死锁（强制覆盖模式）**
在原脚本中，对同一个域名重复运行申请会直接报错卡死。最终优化版加入了智能判断：如果检测到当前域名在系统内已有记录，会自动在底层追加 `--force` 参数，启动强制覆盖更新，方便你随时更换解析商或重新生成密钥。
* **全局证书状态统计与查询**
* **主界面动态统计**：每次打开脚本，主界面都会动态计算并显示当前系统内一共成功托管了多少个有效证书。
* **一键列表查询**：提供专门的菜单选项，可以一键以表格形式罗列出所有已成功申请的域名、密钥长度、创建时间以及下一次自动续期的具体时间点。

### 2. 菜单选项功能一览

| 菜单编号 | 功能名称 | 对应后台底层逻辑 |
| --- | --- | --- |
| **1** | **申请新证书** | 交互式输入域名，检查并匹配本地与解析 IP，选择 80 端口或 DNS API 进行证书签发，最后动态创建独立文件夹并导出重命名。 |
| **2** | **查看所有域名列表** | 调用原版 `acme.sh --list`，完整打印服务器上当前托管的所有证书信息明细。 |
| **3** | **手动强制一键续期** | 忽略 90 天的剩余时间限制，立刻强制让系统内所有已注册的域名在后台进行一次更新排队与文件同步。 |
| **4** | **彻底清空并卸载** | 撤销所有证书，强力清空 `/root/cert` 目录，从系统 `crontab` 中抹除定时任务，并彻底御载 `acme.sh` 核心环境。 |
| **0** | **退出** | 退出当前 Shell 交互界面。 |
